    Fråga 1:
    Vad är CSS?

    • Varför finns CSS?

      • Det finns för att göra programmering av HTML mycket lättare. Dels gör det att man kan återanvända
      den kod man skriver i CSS väldigt lätt men också återanvändas på flera olika webbsidor. HTML var från
      början skapat för att beskriva element inte för att göra design eller layout, detta problemet löser CSS.

    • Hur använder vi oss av CSS?

      • Antingen med hjälp av:

      • External style sheet
      • Internal style sheet
      • Inline style
      External style sheet är ett eget dokument där man kan lägga in all CSS kod för att lätt kunna använda
      den på flera webbsidor.
      Internal style sheet skrivs i metadelen av webbsidans meta information. Detta gör att dem styles man kodar
      endast kan användas på hemsidan den är skriven på.
      Inline style används direkt i elementet på en lina av kod, detta för att man endast vill stylea en väldigt
      specifik sak.

    2. Fråga 2:
    Hur använder vi CSS?

    • Vilka versioner finns det?

      1. w3.css Version 3

        2. Är nu mer den äldre modellen av CSS då version 4 kom 24 mars 2017.
        Denna modellen har exempelvis kvar standardfärger för knappar och även vissa sorters
        klasser som inte är optimala i jämförelse med CSS version 4.

      2. w3.css Version 4

        3. Den senaste uppdateringen av CSS är version 4, den är dubbelt så snabb
        som föregångaren version 3. Den designades för att kunna man skulle kunna skapa en till
        version av CSS, av mobil versionen.

      3. w3mobile.css

        4. Mobilversionen är inriktad mot utvecklingen av applikationer som ska användas i telefoner,
        den följer ungefär samma regler som PRO versionen. Dock är w3mobile.css anpassat till användning
        för telefoner och kan därför inte användas i utveckling inom webbplatser eller applikationer
        som främst ska kunna användas på datorer.

      4. w3pro.css

        5. Pro versionen följer med version 4 och är designad för användning av teman och färgklasser
        som utvecklaren står för själv istället för att de använder sig av de standard som funnits i tidigare
        versioner, exempelvis version 3.
